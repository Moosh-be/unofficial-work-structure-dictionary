# Index (OBJ-001)

L'index constitue le point d'entrée et le sommaire de l'ensemble des objets métier du projet.

Il permet de naviguer, comprendre et relier les différents artefacts entre eux.

---

## Rôle

L'index sert à :

- offrir une **vue d'ensemble** du référentiel
- faciliter la **découverte** des objets métier
- garantir la **cohérence** entre les différentes sections
- servir de **table des matières dynamique**

---

## Structure

L'index répertorie les objets métier selon une classification numérique :

| Code | Objet | Description |
| :--- | :--- | :--- |
| OBJ-001 | Index | Sommaire et navigation |
| OBJ-002 | Livrables | Livrables du projet |
| OBJ-003 | Décisions | Décisions structurantes |
| OBJ-004 | Individus | Personnes impliquées |
| OBJ-005 | Milestones | Jalons clés |
| OBJ-006 | Resources | Ressources allouées |
| OBJ-007 | Stakeholders | Parties prenantes |
| OBJ-008 | Risks | Risques identifiés |
| OBJ-009 | Budgets | Budgets alloués |
| OBJ-010 | Documents | Documentation |
| OBJ-011 | KPI | Indicateurs de performance |
| OBJ-012 | Test Cases | Cas de test |
| OBJ-013 | Requirements | Besoins et exigences |
| OBJ-014 | Calendar | Calendrier |
| OBJ-015 | Assignation | Attribution des rôles |

---

## Utilisation

L'index est un artefact vivant. Il doit être mis à jour à chaque ajout ou modification d'un objet métier.
