# Sprints (OBJ-035)

Les Sprints (ou cycles/agiles) organisent le travail en itérations courtes et planifiées.

---

## Rôle

Les Sprints permettent de :

- **organiser** le travail en itérations
- **planifier** les WorkPackages par cycle
- **suivre** la vélocité et la progression
- **cadencer** les livraisons incrémentales

---

## Cycle de vie d'un Sprint

1. **Planned** : Backlog sélectionné, objectifs définis
2. **Active** : Travail en cours, daily updates
3. **Review** : Revue de sprint, démonstration
4. **Closed** : Rétrospective, burndown complété

---

## Informations d'un Sprint

- **Nom** : Sprint N (ex: Sprint 1, Sprint 2)
- **Durée** : Nombre de jours (généralement 2 ou 4 semaines)
- **Dates** : Début et fin planifiées
- **Objectif** : But du sprint
- **WorkPackages** : Backlog sélectionné
- **Vélocité** : Points réalisés vs planifiés
- **Burndown** : Graphique de progression

---

## Bonnes pratiques

- **Sprint court** : 2 semaines est l'idéal pour débuter
- **Capacité réaliste** : Ne pas surcharger le backlog
- **Objectif clair** : Un sprint doit livrer de la valeur
- **Rétrospective** : Améliorer en continu

---

## Principe

> Un sprint sans objectif clair est un sprint perdu.
> Une vélocité non suivie est une vélocité ignorée.
