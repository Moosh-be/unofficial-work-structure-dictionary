# Risques (OBJ-008) — Contexte PM² v1.0

## Cadre

Ce document présente le contexte **PM²** (European Commission Methodology) pour l'objet métier **Risks (OBJ-008)**.

PM² structure les risques via le **Risk Register** — un artefact obligatoire de la méthodologie européenne.

Les risques PM² s'appuient sur :
- Un registre centralisé (Risk Register)
- Une matrice probabilité × impact systématique
- Une distinction risque/opportunité
- Un lien formel avec les plans de réponse (Risk Response Plan)

## Principes PM² pour les risques

- Tout risque identifié est consigné dans le Risk Register
- La probabilité et l'impact sont évalués systématiquement
- Le score détermine la priorité de traitement
- Chaque risque actif a un plan de réponse documenté
- La révision des risques est périodique (chaque période de reporting)

## Différences avec Moosh

| Aspect | Moosh | PM² |
|--------|-------|-----|
| Structure | Objet autonome | Risk Register (artefact obligatoire) |
| Score | Calculé (probabilité × impact) | Matrice PM² (probabilité × impact × impact business) |
| Opportunité | Non distingué | Distinct (risque positif) |
| Plan réponse | Champ plan_reponse | Risk Response Plan (artefact séparé) |
| Révision | Champ date_revison | Période de reporting fixe |
