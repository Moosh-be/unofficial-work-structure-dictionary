# Incidents (OBJ-022)

Les incidents représentent les événements non planifiés qui perturbent ou pourraient perturber le bon déroulement du projet.

---

## Rôle

Les incidents permettent de :

- **documenter** les perturbations survenues
- **analyser** les causes racines
- **corriger** les effets
- **prévenir** la récurrence

---

## Différence entre incident, risque et problème

- **Risque** : Événement potentiel (futur)
- **Incident** : Événement réel survenu (présent/passé)
- **Problème** : Cause racine identifiée d'un ou plusieurs incidents

---

## Cycle de vie d'un incident

1. **Détection** : L'incident est identifié
2. **Enregistrement** : Description et impact initial
3. **Classification** : Sévérité et priorité
4. **Diagnostic** : Analyse des causes
5. **Résolution** : Actions correctives
6. **Clôture** : Vérification et capitalisation

---

## Niveaux de sévérité

- **Critique** : Blocage total du projet
- **Majeur** : Impact significatif sur les objectifs
- **Mineur** : Impact局部, contournement possible
- **Mineur** : Gêne mineure, non-blocking

---

## Principe

> Un incident non analysé se reproduira.
> Un incident analysé sans correction est une leçon perdue.

---
