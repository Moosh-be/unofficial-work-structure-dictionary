# Gantt (OBJ-040)

Le Gantt est la vue planning qui organise les WorkPackages dans leur ordre temporel et leurs dépendances.

---

## Rôle

Le Gantt permet de :

- **visualiser** le planning global du projet
- **identifier** le chemin critique
- **suivre** les chevauchements et dépendances
- **planifier** les séquences logiques

---

## Éléments du Gantt

Un diagramme de Gantt contient :

- **Barres de temps** : Durée planifiée de chaque WorkPackage
- **Dépendances** : Flèches entre WorkPackages liés
- **Jalons** : Points dans le temps (réalisations)
- **Chemins critiques** : Séquence de tâches déterminant la durée totale
- **Progression** : Avancement réel vs planifié
- **Calendrier** : Jours ouvrés, fériés, contraintes

---

## Fonctionnalités

- **Dépendances** : FS, SS, FF, SF
- **Décalages (lag)** : Retard ou avance entre tâches
- **Chemins critiques** : Calcul automatique du délai minimal
- **Niveaux de zoom** : Mois, semaine, jour
- **Drag & drop** : Réorganisation visuelle des dates
- **Nœuds WBS** : Vue hiérarchique par lots

---

## Métriques du Gantt

- **Durée du projet** : Date fin planifiée
- **Chemin critique** : Tâches déterminantes
- **Floats/slacks** : marges de manœuvre
- **Ressources** : Allocation sur le temps
- **Bougies** : Points de contrôle

---

## Principe

> Un Gantt sans dépendances est un tableau de dates.
> Un Gantt non mis à jour est un Gantt faux.
