# Time Entries (OBJ-027)

Les Time Entries (saisies de temps) enregistrent le temps passé sur chaque WorkPackage par chaque membre de l'équipe.

---

## Rôle

Les Time Entries permettent de :

- **tracer** le temps réellement investi
- **comparer** estimé vs réel
- **facturer** les prestations (si applicable)
- **analyser** la productivité et les dérives

---

## Fonctionnement

Chaque saisie de temps enregistre :

- **Qui** a travaillé (membre de l'équipe)
- **Sur quoi** (WorkPackage)
- **Combien de temps** (heures/jours)
- **Quand** (date de travail)
- **Pourquoi** (description du travail effectué)
- **Quel type** de travail (développement, réunion, bugfix, etc.)

---

## Règles de saisie

- **Régularité** : Saisie quotidienne ou hebdomadaire recommandée
- **Précision** : À la demi-heure ou à l'heure près
- **Justification** : Description du travail effectué pour chaque saisie
- **Contrôle** : Vérification par le responsable d'équipe

---

## Utilisation

Les Time Entries sont utilisées pour :

- **Rapports de temps** : Par individu, équipe, projet
- **Suivi budgétaire** : Coût réel vs estimé
- **Analyse de productivité** : Vitesse d'exécution, écarts
- **Facturation** : Si le projet est en mode facturation au temps passé

---

## Principe

> Un temps non saisi est un temps perdu pour le projet.
> Un temps non vérifié est un temps suspect.
