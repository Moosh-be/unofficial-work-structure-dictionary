# Requirements (OBJ-013)

Les requirements (besoins et exigences) représentent ce que le projet doit produire ou accomplir.

C'est le fondement sur lequel tout le projet est construit.

---

## Rôle

Les exigences permettent de :

- **définir clairement** ce qui est attendu
- **servir de base** à la validation (tests)
- **piloter les changements** (gestion de configuration)
- **garantir** l'alignement avec les attentes

---

## Classification

Les exigences peuvent être classées en :

- **Fonctionnelles** : Ce que le système doit faire
- **Non-fonctionnelles** : Comment le système doit le faire (qualité, performance, sécurité)
- **Contraintes** : Restraints imposés (techniques, réglementaires, budgétaires)
- **Exigences métier** : Objectifs stratégiques du projet

---

## Niveaux de granularité

- **Besoins métier** (Level 1) : Objectifs stratégiques
- **Exigences fonctionnelles** (Level 2) : Fonctionnalités attendues
- **Exigences détaillées** (Level 3) : Spécifications précises

---

## Principes

> Un besoin non documenté est un besoin non satisfait.
> Une exigence non tracée est une exigence non validée.
---

## Normes existantes

- **ISO/IEC/IEEE 29148** — Standard pour la gestion des exigences (requirements engineering)
- **BABOK (IIBA)** — Guide de l'analyse d'affaires (Business Analysis)
- **PMBOK® Guide** — Scope management et traçabilité des exigences
- **INCOSE** — Guide de l'ingénierie système (systems engineering requirements)
