# Risques (OBJ-008) — Contexte Moosh v1.0

## Cadre

Ce document présente le contexte **Moosh** pour l'objet métier **Risks (OBJ-008)**.

Moosh est le contexte personnel de l'auteur du Work Structure Dictionary. Il représente une approche empirique, issue de ~30 ans d'expérience terrain sur des projets de natures variées.

## Principes Moosh pour les risques

Un risque identifié et géré est un problème évité. Un risque non identifié est une surprise.

La gestion des risques n'est pas une corvée administrative — c'est un levier de pilotage proactif. Anticiper permet de préparer des plans de mitigation et d'exploiter les opportunités.

## Objectif de ce contexte

Dans le contexte Moosh v1.0, un objet Risk encode :
- Son identification (id, titre, description, catégorie)
- Son analyse (probabilité, impact, score calculé)
- Sa réponse planifiée (éviter, atténuer, transférer, accepter)
- Son suivi (responsable, statut, dates)
- Sa réalisation éventuelle (date réelle, lien vers incident)

## Différence avec un incident

Un **risque** est un événement **incertain** qui pourrait survenir.
Un **incident** est un événement **réel** qui s'est produit.

La frontière est claire : un risque réalisé doit être relié à un incident.
