# Analyses — Partie "organique" du projet

Ce dossier est la partie **organique** du Work Structure Dictionary.

## Différence avec les normes

Les normes (01-normes-communes/, 02-Objets-Metier/) sont froides, tranchées, le résultat.
Les analyses sont l'exploration, la réflexion, le "pourquoi" et le "comment".

> La norme dit "comment". L'analyse dit "pourquoi".

## Ce qu'on y trouve

- **Pourquoi** ce référentiel existe
- **Analyse des existants** : outils, méthodes, normes — qu'est-ce qui existe déjà ?
- **Démarche** : comment on s'y prend, comment on progresse
- **Contextes** : chaque auteur/autorité propose sa propre variante d'un objet métier

## Pourquoi des analyses ?

Parce que :
- Un registre des risques, pourquoi ? Pas juste "comment le remplir"
- Une décision, comment identifier qu'un risque est un risque et pas un incident ?
- La méthodologie, ce n'est pas une méthode de plus — c'est la réflexion sur comment approcher la gouvernance

## Contextes

Chaque contexte = une autorité qui décide "la norme, c'est comme ça".

Voir `04-contextes/README.md` pour la liste et les données de chaque contexte.
