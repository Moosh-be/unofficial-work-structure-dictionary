# Files (OBJ-032)

Les Files (fichiers) gèrent les pièces jointes et documents attachés aux WorkPackages et au projet.

---

## Rôle

Les Files permettent de :

- **attacher** des fichiers aux WorkPackages
- **organiser** la documentation projet
- **versionner** les documents
- **contrôler** l'accès aux fichiers sensibles

---

## Fonctionnalités

- **Attachement** : Joindre des fichiers à n'importe quel WorkPackage
- **Organisation** : Dossier et métadonnées
- **Contrôle d'accès** : Permissions par fichier
- **Historique** : Versionnement des documents
- **Recherche** : Indexation du contenu

---

## Types de fichiers courants

- **Documents** : Spécifications, rapports, plans
- **Dessins** : Schémas, diagrams, maquettes
- **Données** : Fichiers de configuration, exports
- **Images** : Captures d'écran, photos
- **Code** : Extrait de code, patches
- **Archives** : Packages de livraison

---

## Règles de gestion

- **Taille maximale** : Limite par fichier et par projet
- **Types autorisés** : Liste blanche ou noire d'extensions
- **Versionnement** : Conservation des anciennes versions
- **Traçabilité** : Qui a uploadé, quand, sur quel WorkPackage

---

## Principe

> Un WorkPackage sans pièce jointe est un WorkPackage incomplet.
> Un fichier sans metadata est un fichier perdu.
