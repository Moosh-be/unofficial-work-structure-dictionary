# Lessons Learned (OBJ-023)

Les retours d'expérience (lessons learned) capturent les enseignements tirés des réussites et erreurs du projet.

---

## Rôle

Les lessons learned permettent de :

- **capitaliser** sur les expériences vécues
- **éviter** de répéter les mêmes erreurs
- **répliquer** les bonnes pratiques
- **améliorer** en continu les processus

---

## Quand collecter ?

- **Régulièrement** : En réunion d'équipe, après chaque étape majeure
- **Après les incidents** : Analyse post-mortem
- **Aux gates** : Bilan de phase
- **En clôture** : Retour global sur le projet

---

## Classification des enseignements

- **Bonnes pratiques** : Ce qui a bien fonctionné à répliquer
- **Points d'attention** : Ce qui a posé problème à éviter
- **Erreurs à ne plus reproduire** : Leçons tirées des incidents
- **Idées d'amélioration** : Suggestions pour les prochains projets

---

## Principe

> Un projet sans retour d'expérience est un projet qui gaspille son expérience.
> Un retour d'expérience non partagé est un retour d'expérience perdu.

---

## Normes existantes

- **PMBOK® Guide** — Lessons learned register et knowledge management
- **ISO 21502** — Capitalisation et amélioration continue
- **PRINCE2** — Lessons practice à chaque étape
- **ISO 30401** — Knowledge management system standard
