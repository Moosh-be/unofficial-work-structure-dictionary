# Calendar (OBJ-014)

Le calendrier représente la structure temporelle du projet.

Il organise les dates clés, les périodes de travail et les contraintes temporelles.

---

## Rôle

Le calendrier permet de :

- **structurer** le planning du projet
- **définir** les périodes de travail
- **gérer** les contraintes temporelles
- **coordination** entre les équipes et parties prenantes

---

## Composantes

Un calendrier projet comprend :

- **Période de référence** : Dates de début et fin du projet
- **Jours ouvrés** : Jours de travail habituels
- **Jours fériés** : Jours non ouvrables
- **Contraintes** : Périodes de indisponibilité, fenêtres d'intervention
- **Événements** : Réunions, revues, livraisons planifiées

---

## Niveaux de granularité

- **Stratégique** : Jalons majeurs, phases
- **Tactique** : Sprints, itérations, cycles
- **Opérationnel** : Planification hebdomadaire, affectations

---

## Principes

> Un calendrier non partagé est un calendrier ignoré.
> Un planning non mis à jour est un planning faux.
