# Test Cases (OBJ-012)

Les test cases (cas de test) représentent l'ensemble des scénarios de validation du projet.

Ils permettent de vérifier que les livrables satisfont les exigences et besoins définis.

---

## Rôle

Les tests permettent de :

- **vérifier** la conformité aux exigences
- **détecter** les défauts et non-conformités
- **garantir** la qualité des livrables
- **documenter** la couverture de validation

---

## Classification

Les tests peuvent être classés par type :

- **Tests unitaires** : Validation des composants individuels
- **Tests d'intégration** : Validation des interfaces entre composants
- **Tests fonctionnels** : Validation des fonctionnalités attendues
- **Tests de non-fonctionnels** : Performance, sécurité, ergonomie
- **Tests d'acceptation** : Validation par le client/utilisateur

---

## Structure d'un test case

Un test case typique contient :

- **Préconditions** : État requis avant exécution
- **Étapes** : Séquence d'actions à effectuer
- **Données de test** : Données d'entrée
- **Résultat attendu** : Comportement attendu
- **Résultat obtenu** : Résultat réel (après exécution)
- **Statut** : Passé, Échoué, Bloqué, Non exécuté

## Normes existantes

- **ISO/IEC/IEEE 29118** — Standard pour le test de logiciels
- **ISO/IEC/IEEE 29147** — Test case design et traçabilité
- **ISTQB** — Certified tester syllabus (niveaux Foundation, Advanced)
- **PMBOK® Guide** — Quality assurance et contrôle qualité via tests
