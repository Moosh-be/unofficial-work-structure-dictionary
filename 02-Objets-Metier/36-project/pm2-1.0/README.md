# Project (OBJ-036) — Contexte PM² v1.0

## Cadre

Ce document présente le contexte **PM²** (European Commission Methodology) pour l'objet métier **Project (OBJ-036)**.

PM² structure les projets via une hiérarchie formelle :
- **Portefeuille** → **Programme** → **Projet**
- Chaque niveau a des artefacts dédiés (Programme Charter, Project Charter)
- Les configurations sont définies dans le Project Charter

## Principes PM² pour les projets

- Un projet PM² est démarré par un Charter signé
- Le périmètre est formalisé dans le Charter
- Le projet a un budget alloué (ligne budgétaire dédiée)
- Le project manager est nommé formellement
- Le projet est piloté par des jalons (milestones) et des reports

## Différences avec Moosh

| Aspect | Moosh | PM² |
|--------|-------|-----|
| Démarrage | Création libre | Project Charter signé obligatoirement |
| Hiérarchie | parent_id explicite | Portefeuille → Programme → Projet (formel) |
| Périmètre | Description libre | Charter document formel |
| Budget | Champ budget_total | Ligne budgétaire dédiée |
| Pilotage | Santé JSON | Jalons + reports périodiques |
