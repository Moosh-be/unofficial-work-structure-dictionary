# Reports (OBJ-041)

Les Reports (rapports) produisent des synthèses visuelles et analytiques sur l'état du projet.

---

## Rôle

Les Reports permettent de :

- **synthétiser** l'état du projet
- **visualiser** les données clés
- **communicer** aux parties prenantes
- **piloter** par les chiffres

---

## Rapports classiques

| Rapport | Description | Fréquence |
|---------|-------------|-----------|
| **Burndown** | Avancement vs temps restant | Hebdo |
| **Burnup** | Travail complété vs total | Hebdo |
| **Vélocité** | Points réalisés par sprint | Par sprint |
| **Statut des WorkPackages** | Répartition par statut | Hebdo |
| **Budget vs Réel** | Dépenses prévisionnelles vs effectives | Mensuel |
| **Risques** | Risques actifs, réalisés, inhibés | Hebdo |
| **Temps passés** | Heures travaillées par individu/équipe | Mensuel |
| **Avancement global** | Pourcentage complété du projet | Hebdo/Mensuel |
| **Gantt view** | Planning global | À la demande |
| **Cohérence** | Qualité des données projet | À la demande |

---

## Types de visualisation

- **Graphiques en barres** : Comparaisons, répartition
- **Graphiques en courbes** : Évolution dans le temps
- **Camemberts / Donuts** : Proportions
- **Tableaux** : Données détaillées
- **Jauges** : Indicateurs de performance
- **Heatmaps** : Intensité, densité

---

## Automatisation

- **Programmation** : Rapports générés automatiquement
- **Distribution** : Envoi par email à intervalles réguliers
- **Templates** : Modèles réutilisables
- **Export** : PDF, Excel, PNG

---

## Principe

> Un projet sans rapport est un projet aveugle.
> Un rapport non lu est un rapport inutile.
