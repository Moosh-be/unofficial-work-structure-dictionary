# Décisions (OBJ-003) — Contexte PM² v1.0

## Cadre

Ce document présente le contexte **PM²** (European Commission Methodology) pour l'objet métier **Décisions (OBJ-003)**.

PM² structure les décisions via le **Decision Log** — un artefact obligatoire de la méthodologie européenne.

Les décisions PM² s'appuient sur :
- Un registre centralisé des décisions (Decision Log)
- Une classification par instance (COPIL, CC, PM, Equipe)
- Un lien formel avec les Change Requests
- Une évaluation d'impact systématique (coût, délai, qualité)

## Principes PM² pour les décisions

- Toute décision structurante est consignée dans le Decision Log
- Les décisions sont classées par instance décisionnaire
- Chaque décision impacte au moins un triptyque coût/délai/qualité
- Les décisions de COPIL sont tracées formellement
- Les décisions d'équipe sont simplifiées mais enregistrées

## Différences avec Moosh

| Aspect | Moosh | PM² |
|--------|-------|-----|
| Structure | Objet autonome | Decision Log (artefact obligatoire) |
| Workflow | Identifié → Actif → Réalisé/Inhibé/Clôturé | Draft → Review → Approved/Rejected |
| Classification | Type (Stratégique/Tactique/Opérationnelle) | Instance (COPIL, CC, PM, Equipe) |
| Impact | Champ impact_cout/delai séparés | Triptyque coût/délai/qualité systématique |
| Lien CR | Relation libre | Change Request lié (obligatoire si décision structurante) |
